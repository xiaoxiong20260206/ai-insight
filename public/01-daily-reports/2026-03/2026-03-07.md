## 一、大模型 (Foundation Models)

### 1. 最近动态

#### 🌏 海外

- 🔴 **[Anthropic研究：AI劳动力市场影响的新度量与早期证据](https://www.anthropic.com/research/labor-market-impacts)** - Anthropic · 3月6日
  - Anthropic引入"observed exposure（实际曝光度）"新指标，结合理论LLM能力和真实使用数据。关键发现：计算机程序员以75%任务覆盖率位居最受影响职业首位，客服代表和数据录入员紧随其后。
  - **为什么重要**: 尽管AI理论上可覆盖94%的计算机/数学类任务，但实际覆盖仅33%——AI远未达到理论上限。更关键的是，22-25岁年轻人进入高曝光职业的比率已下降约14%。

- 🔴 **[Anthropic与战争部争端升级：CEO称准备起诉](https://www.businessinsider.com/anthropic-ceo-dario-amodei-pentagon-lawsuit-conversations-2026-3)** - Business Insider · 3月5日
  - Dario Amodei发布声明确认收到战争部的"清单"信，但强调Anthropic将坚持安全底线。此前Anthropic已拒绝了国防部的多项合作要求。
  - **背景**: 这是AI行业"安全 vs 国防应用"核心分歧的标志性事件

- 🟡 **[OpenAI发布GPT-5.4 Thinking安全评估报告](https://openai.com/index/gpt-5-4-thinking-system-card/)** - OpenAI · 3月5日
  - 详细披露GPT-5.4 Thinking的安全评估结果，包括在推理、编码和Agent工作流中的行为表现。注意：不存在"GPT-5.3 Thinking"模型，5.4是首个结合推理链的5系列模型。

### 2. 热门观点

> **Anthropic 劳动力市场研究报告**
> "AI远未达到其理论能力：实际覆盖仍是理论可行范围的一小部分。我们未发现高度曝光的工人群体出现系统性失业增加，但有初步迹象表明年轻工人进入这些职业的速度正在放缓。"
> 
> **解读**: 这份研究的意义在于提供了一个可持续追踪的框架——不是"AI会不会取代人"的简单二分法，而是"哪些任务在什么程度上被自动化"的精细度量。75%覆盖率的程序员并没有大规模失业，说明"任务替代≠岗位替代"。
> 来源: [Anthropic Research](https://www.anthropic.com/research/labor-market-impacts)

### 3. 趋势洞察

**"理论能力-实际使用"的巨大鸿沟是当前AI就业讨论的核心盲区**: Anthropic的数据显示，即使是计算机/数学类最受影响的领域，实际AI覆盖率也只有33%（理论可达94%）。这意味着短期内AI更多是增强而非替代工作，但长期趋势不可忽视——22-25岁年轻人的就业信号值得持续关注。

**建议**: 企业在制定AI转型计划时，应区分"理论可自动化任务"和"实际被自动化任务"。不要因为AI理论上能做某件事就急于裁员；也不要因为短期没看到大规模失业就忽视准备。

## 二、AI Coding

### 1. 最近动态

#### 🌏 海外

- 🔴 **[GitHub Copilot VS Code v1.110发布 + Figma MCP Server集成](https://github.blog/changelog/2026-03-06-github-copilot-in-visual-studio-code-v1-110-february-release/)** - GitHub · 3月6日
  - GitHub Copilot发布VS Code v1.110版本，最重要的更新是**Figma MCP Server正式上线**——AI编程工具可以直接连接Figma设计稿，在代码生成时理解设计意图。
  - **为什么重要**: 这打通了设计→代码的最后一环。设计师在Figma中的组件、样式、布局信息可以直接被AI理解和使用，而不是靠开发者人工解读。

- 🟡 **[Omniflow AI发布：解决从规范到生产的断裂](https://www.blufftontoday.com/press-release/story/57803/former-uber-engineering-leader-launches-omniflow-ai-to-solve-the-spec-to-production-crisis/)** - EINPresswire · 3月6日
  - 由前Uber工程负责人创立，Omniflow AI定位为"给创始人和产品团队构建生产级软件的AI平台"。直接瞄准当前vibe coding最大痛点：AI写的代码和生产级代码之间的鸿沟。
  - **定位**: 从"AI帮你写代码"走向"AI帮你交付生产级软件"

#### 🛠️ 实践

- 🟡 **[Figma→代码工具链2026年全景](https://javascript.plainenglish.io/best-figma-to-code-tools-for-2026-5e8063b64280)** - JavaScript in Plain English · 持续更新
  - Figma MCP的意义不止于GitHub Copilot——它让ChatGPT、Claude等所有支持MCP的AI模型都能"看到"设计稿。这是设计工具从"导出资源"到"语义理解"的质变。

### 2. 热门观点

> **GitHub Blog · Figma MCP Server发布**
> "GitHub Copilot用户现在可以连接Figma MCP服务器，让AI在编码时直接理解设计上下文。"
> 
> **解读**: MCP（Model Context Protocol）正在成为AI工具集成的标准协议。Figma支持MCP意味着设计资产不再是"截图+口头描述"，而是结构化的、AI可理解的上下文。这将显著减少设计到开发的交接成本。
> 来源: [GitHub Blog](https://github.blog/changelog/2026-03-06-github-copilot-in-visual-studio-code-v1-110-february-release/)

### 3. 趋势洞察

**MCP协议正在重塑开发工具链的集成方式**: Figma MCP Server不是孤立事件——它代表着"工具通过MCP向AI开放语义接口"的趋势。预计接下来会看到更多设计工具（如Sketch、Adobe XD）、项目管理工具（如Linear、Jira）、文档工具（如Notion）提供MCP接口。

**建议**: 如果你是工具开发者，现在开始考虑提供MCP接口。如果你是使用者，关注你常用工具的MCP支持情况——这将决定AI能在多大程度上理解你的工作上下文。

## 三、AI 应用

### 1. 最近动态

#### 🌏 海外

- 🔴 **[ChatGPT for Excel + 金融数据集成正式发布](https://openai.com/index/chatgpt-for-excel/)** - OpenAI · 3月6日
  - OpenAI发布ChatGPT Excel插件，用户可以直接在Excel中使用AI进行金融建模、数据提取和分析。同时集成多个金融数据源，让ChatGPT可以实时调用市场数据。
  - **为什么重要**: Excel是全球数亿知识工作者的核心工具。ChatGPT从"对话框"走进"电子表格"，意味着AI正在嵌入实际的工作流程，而不是作为独立工具存在。

- 🔴 **[Figma发布GitHub Copilot to Figma新功能](https://www.figma.com/release-notes/)** - Figma · 3月6日
  - 设计师在Figma中可以直接使用GitHub Copilot的能力，反向打通代码→设计的链路。同时AI功能在Figma中持续增强。
  - **双向链路**: Figma→Code（MCP Server）+ Code→Figma（Copilot to Figma）形成完整闭环

- 🟡 **[OpenAI推出"Adoption"新闻频道](https://openai.com/index/introducing-the-adoption-news-channel/)** - OpenAI · 3月6日
  - OpenAI推出专注企业AI采纳的新业务博客频道，标志着OpenAI从面向开发者到面向企业决策者的战略延伸。

### 2. 热门观点

> **OpenAI · ChatGPT for Excel发布声明**
> "大多数组织仍然将AI作为一系列孤立的试点项目来管理。ChatGPT for Excel让AI能力直接嵌入现有的数据处理、应用程序和运营模型中。"
> 
> **解读**: OpenAI的策略非常清晰——不要让用户"打开ChatGPT做事"，而是让ChatGPT在用户"已经在做的事情里"出现。Excel集成是这一策略的完美体现：数亿用户无需改变习惯就能使用AI。
> 来源: [OpenAI](https://openai.com/index/chatgpt-for-excel/)

### 3. 趋势洞察

**AI从"独立应用"到"嵌入式能力"的范式转变**: ChatGPT for Excel不是偶然——此前ChatGPT已经集成了Slack、Google Drive、Notion等工具。这标志着AI应用的下一阶段：不是"人去找AI"，而是"AI在你的工具里等你"。这对所有SaaS产品都是挑战——如果你的产品不集成AI，你的用户会在有AI的竞品那里完成工作。

**风险提示**: 设计→代码的双向打通（Figma MCP + Copilot to Figma）可能加速"低代码/无代码"趋势，对初级前端开发人员的需求产生影响。

## 四、AI 行业 (Industry & Investment)

### 1. 最近动态

#### 🌏 投融资

- 🔴 **[Anduril融资$40亿，估值$600亿](https://techcrunch.com/2026/03/03/anduril-aims-at-60-billion-valuation-in-new-funding-round/)** - TechCrunch · 3月3日
  - 由Palmer Luckey领导的防务AI巨头Anduril正在进行约$40亿的新一轮融资，估值$600亿，由Thrive Capital和a16z领投。估值较上轮（2024年6月约$300亿）翻倍。
  - **为什么重要**: 这是AI在"物理世界"（国防、自动驾驶、机器人）领域估值最高的公司之一，标志着AI资本从纯软件向物理世界延伸。

- 🔴 **[OpenAI前CRO Bob McGrew创立Arda，融$7000万做自主工厂AI](https://seekingalpha.com/news/4560788-ex-openai-research-chief-seeks-70m-for-ai-factory-automation-startup-wsj)** - WSJ · 3月4日
  - OpenAI前首席研究官Bob McGrew正在为其新创公司Arda筹集约$7000万资金，由Founders Fund、Accel、Khosla Ventures投资。Arda旨在构建帮助运营自主工厂的软件平台。
  - **为什么重要**: 顶级AI人才从"模型公司"流向"应用公司"的信号——AI的下一个战场是物理世界的自动化。

- 🟡 **[YC支持的Denki融$410万：AI金融审计自动化](https://news.crunchbase.com/venture/yc-backed-denki-raise-financial-audit-automation-ai/)** - Crunchbase · 3月5日
  - 由两位20多岁的兄弟创立的Denki，用AI自动化上市公司的内部审计工作（计划、测试、文档），用99%软件+1%人工的模式替代传统审计流程。
  - **关注点**: 审计是典型的"高薪+高重复"工作，正好处于Anthropic研究所指出的"高曝光"区间

#### 📊 行业动态

- 🟡 **[Anthropic与战争部争端持续发酵](https://www.americanprogress.org/article/the-department-of-defenses-conflict-with-anthropic-and-deal-with-openai-are-a-call-for-congress-to-act/)** - American Progress · 3月5日
  - 美国进步中心发文呼吁国会立法，认为战争部与Anthropic的冲突以及与OpenAI的合作凸显了AI军事应用缺乏监管框架的问题。
  - **行业影响**: 这场争端正在迫使AI公司明确立场——"安全优先"还是"服从政府"

### 2. 热门观点

> **WSJ · Bob McGrew创立Arda**
> "Bob McGrew正在筹集$7000万资金，创建一个帮助运营自主工厂的软件平台。"
> 
> **解读**: OpenAI的前CRO选择"工厂AI"而非"下一个聊天机器人"，这本身就是一个强烈信号。AI的真正价值不在于生成文字和图片，而在于优化物理世界的运营。从McGrew离开OpenAI到创立Arda，我们看到的是AI从"认知层"向"物理层"的战略转移。
> 来源: [WSJ](https://www.wsj.com/tech/ai/openais-former-research-chief-aims-to-automate-manufacturing-with-ai-8871f265)

### 3. 趋势洞察

**AI资本正在从"软件世界"流向"物理世界"**: Anduril（$600亿防务AI）和Arda（自主工厂）代表同一趋势——AI的下一个增长极不在屏幕上，而在工厂里、战场上、马路上。这与OpenAI近期将公司重心转向Agent和工具使用的方向一致。

**建议**: 关注AI在制造、物流、能源等物理行业的落地进展。这些领域的AI应用虽然不如聊天机器人引人注目，但商业价值可能更大、壁垒更高。

## 五、企业 AI 转型 (Enterprise Transformation)

### 1. 最近动态

#### 🌏 海外实践

- 🔴 **[OpenAI发布"五大AI价值模型"：企业AI采纳方法论](https://openai.com/index/the-five-ai-value-models-driving-business-reinvention/)** - OpenAI · 3月5日
  - OpenAI推出企业AI采纳框架，指出"大多数组织仍将AI作为一系列孤立试点来管理"。五大价值模型帮助企业从零散试点升级为系统化部署。
  - **为什么重要**: 这是OpenAI首次以"企业方法论"而非"技术产品"的形式发布内容，标志着其从技术公司向平台公司转型。

- 🔴 **[Anthropic研究揭示AI对白领就业的精细影响](https://www.anthropic.com/research/labor-market-impacts)** - Anthropic · 3月6日
  - 最受AI影响的十大职业包括：计算机程序员（75%覆盖）、客服代表、数据录入员、金融分析师等。受影响最大的群体更可能是：年龄较大、女性、受教育程度更高、收入更高的工人。
  - **关键数据**: 高曝光群体收入比低曝光群体高47%，研究生学位占比高近4倍
  - **企业启示**: 转型计划需要关注高收入高技能员工的再培训，而非仅聚焦低层级岗位

#### 🌏 重要预告

- 🟡 **[NVIDIA GTC 2026下周开幕：物理AI和Agent AI成焦点](https://www.nvidia.com/gtc/)** - NVIDIA · 3月16-19日
  - Jensen Huang主题演讲将展示AI在物理AI、Agentic AI、推理和AI工厂等领域的突破。超过1000场分会场。
  - **关注点**: 物理AI（机器人、自动驾驶）和AI推理基础设施将是核心议题

- 🟡 **[Microsoft Copilot Chat预览版即将发布](https://news.microsoft.com/ignite-2025-book-of-news/)** - Microsoft · 预计3月
  - Copilot Chat将在Outlook、Word、Excel和PowerPoint中全面扩展AI能力，从Ignite 2025公布的计划来看，3月将进入预览阶段。

### 2. 热门观点

> **OpenAI · 五大AI价值模型**
> "大多数组织仍将AI作为一系列孤立的试点来管理。"
> 
> **解读**: OpenAI观察到的"孤立试点"问题非常普遍——大量企业在"某个部门试了一下AI"的阶段停滞不前。五大价值模型框架试图解决的正是"从试点到规模化"的鸿沟。值得注意的是，OpenAI开始输出方法论而非仅输出产品，说明企业AI的瓶颈已经从"技术能不能用"转向"组织怎么用"。
> 来源: [OpenAI](https://openai.com/index/the-five-ai-value-models-driving-business-reinvention/)

### 3. 趋势洞察

**企业AI转型的核心矛盾正在从"技术问题"转向"组织问题"**: OpenAI发布方法论、Anthropic发布就业影响研究、德勤以"AI×组织"为主题——三方信号一致：技术已经就绪，瓶颈在于组织如何适应。

**建议**: 
1. 优先阅读OpenAI的"五大AI价值模型"框架，评估你的组织处于哪个阶段
2. 参考Anthropic的就业影响数据，提前识别你的团队中哪些岗位处于"高曝光"区间
3. 制定再培训计划——高收入高技能岗位（程序员、分析师）比低层级岗位更需要关注
