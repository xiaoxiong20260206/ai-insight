  > 详细的 Spec-driven + Claude Code 工作流实操案例，从定义 specs 到生成代码再到自动化测试的完整流程。

### 2. 热门观点

> **Ankit Jain** · Aviator CEO / Latent Space 特约
> "Human-written code died in 2025. Code reviews will die in 2026. Specs become the source of truth. Code becomes an artifact."
> *—— [Latent Space](https://latent.space/p/reviews-dead) · 2026-03-03*

> **Simon Willison** · Django 联合创始人
> Claude Import Memory 功能本质是一段精心构造的 prompt——最好的产品设计可以极其简单。Claude 登顶 App Store 证明了产品品质的力量。
> *—— [simonwillison.net](https://simonwillison.net/2026/Mar/1/claude-import-memory/) · 2026-03-01*

### 3. 趋势洞察

AI 编程工具的竞争焦点已从"生成代码"转向**"自主测试+自动修复"**。Cursor 的 Cloud Agents with Computer Use 能自己运行软件、测试改动、演示功能——这意味着 AI 不只写代码，还能验证代码是否正确。Bugbot Autofix 则进一步实现"发现 Bug → 自动修复 → 自动测试"的闭环。Codex 的多 Agent 分叉、ChatGPT 的交互式代码块，各家都在向**端到端自动化**冲刺。

**💡 本周可以做：** 如果你用 Cursor，打开 Bugbot Autofix 功能试跑一个 PR——评估它的修复质量能否降低你的 Code Review 负担。同时读一下 Heeki Park 的 Spec-driven + Claude Code 实操文章，在一个小项目上试跑。

**⚠️ 值得警惕：** "Code Review 已死"的说法虽然引人注目，但 Spec-driven 的前提是你能写清楚 Spec。对于复杂系统、遗留代码、安全关键场景，人工审查短期内不可替代。不要因为"AI 能写代码"就裁掉 Code Reviewer，而是让他们转型为 Spec Writer。

### 3. 趋势洞察
