🎯 三足鼎立的Agent格局

2026年的AI Agent市场正在经历一场范式转移——从「会话式助手」向「常驻型智能体」演进。 三大平台代表了三种截然不同的哲学：OpenClaw走生态扩张路线，Hermes Agent专注自我进化， Anthropic KAIROS押注云端托管。它们分别回答了同一个问题的不同侧面： 一个真正有用的AI Agent应该是什么样子？

🦅 Hermes Agent：学习型Agent的典范
📝 核心定位

Hermes Agent由Nous Research（开源AI研究实验室）于2026年2月发布， 其核心命题是：如果一个Agent能够从经验中学习并自我改进，会发生什么？ 它不仅仅是一个工具，而是一个"与你共同成长的Agent"。

💡 关键洞察

Hermes的差异化在于内置学习循环（learning loop）——它能从经验中创建技能、在使用过程中改进技能、 主动提醒自己要持久化知识、搜索过往对话，并在多次会话中构建对你的深度理解。

⚙️ 技术架构
  • AIAgent执行循环为核心：与OpenClaw的Gateway控制平面不同，Hermes将Agent循环本身作为核心同步编排引擎
  • 分层记忆系统：FTS5全文搜索 + SQLite持久化 + LLM驱动的会话摘要，支持跨会话召回
  • 自主技能创建：完成复杂任务后，Agent自动编写结构化技能文档，记录程序、陷阱和验证步骤
  • 自训练循环：集成Atropos RL框架，可生成批量轨迹并训练Agent行为
  • 六终端后端：local、Docker、SSH、Daytona、Singularity、Modal，支持serverless休眠
🎯 关键特性
多平台接入

Telegram、Discord、Slack、WhatsApp、Signal、Email、CLI——单一网关进程支持全平台

模型无关

支持Nous Portal、OpenRouter(200+模型)、OpenAI、Kimi、MiniMax等，一键切换

子Agent并行

生成隔离子Agent进行并行工作流，通过RPC调用工具，零上下文成本

TUI终端界面

多行编辑、斜杠命令自动补全、会话历史、流式工具输出

🧠 Skills系统：程序性记忆

OpenClaw的Skills主要由人工编写，而Hermes的Skills是自主生成的。 它们是Agent的"程序性记忆"——每个技能捕获一个经过验证的问题解决方法，包括：

  • 程序步骤和最佳实践
  • 已知陷阱和边界情况
  • 验证和回滚步骤
  • 兼容agentskills.io开放标准
🔬 研究级能力

Hermes不仅是一个产品，更是一个研究平台。它支持批量轨迹生成、Atropos RL环境、 轨迹压缩——用于训练下一代工具调用模型。这反映了Nous Research作为模型训练实验室的基因， 而非单纯的产品公司。

🦀 OpenClaw：生态型Agent的霸主
📝 核心定位

OpenClaw由奥地利开发者Peter Steinberger于2025年底作为周末项目启动（原名Clawdbot）， 迅速成为GitHub增长最快的开源项目之一，截至2026年4月已获得34.5万stars。 2026年2月，Steinberger宣布加入OpenAI，OpenClaw将移交独立基金会运营。

"OpenClaw让'个人自托管Agent'这个想法变得可行。它解决了开发者一直在等待有人解决的问题： 一个能连接到你已经在使用的消息应用的自托管AI Agent。"
— The New Stack
🏗️ 架构特点

OpenClaw采用Gateway控制平面架构——一个单一的长运行进程拥有会话、路由、 工具执行和状态。所有数据流都经过它。这与Hermes的"Agent循环为核心"形成鲜明对比。

  • Gateway作为控制平面：集中式会话管理和路由
  • 跨平台消息网关：单一进程支持多平台同时在线
  • 模型无关设计：Anthropic、OpenAI、Google、Ollama本地模型无缝切换
  • 技能插件系统：workspace、personal、shared、plugins四级作用域
📦 ClawHub生态

ClawHub是OpenClaw的公开技能仓库，拥有超过13,000个社区构建的技能。 它被比作"npm早期时代"——发布门槛低（仅需一周GitHub账号），但安全问题也随之而来。

⚠️ 安全事件

Koi Security审计发现ClawHub上341个恶意技能（335个来自单一ClawHavoc攻击活动）； CVE-2026-25253（CVSS 8.8）涉及不安全的WebSocket自动连接行为； Microsoft建议将其运行时视为"可能被不可信输入影响"；Cisco称其为"安全噩梦"。

OpenClaw已与VirusTotal合作扫描上传技能，并增加了安全指导，但信任模型仍在改进中。

✨ killer feature：跨渠道持久化

开发者可以在工作站上启动任务，晚餐时在Telegram上收到完成通知，然后从手机发送后续指令。 这种跨渠道持久化是OpenClaw病毒式传播的最核心驱动力——它证明了人们想要的Agent 是能够超越浏览器标签页生命周期的存在。

⏰ KAIROS：云端常驻Agent的未来
🔮 泄露事件与曝光

2026年3月底，Claude Code的完整源码通过npm包中的.map文件泄露，有人将其上传至GitHub。 这次泄露揭示了Anthropic正在秘密开发的多个未发布功能，其中最引人注目的是KAIROS—— 一个始终在线的自治Agent，能够发送推送通知并监控GitHub PR。

💡 泄露发现

Reddit用户通过分析泄露代码发现："kairos: an autonomous agent that can send push notifications and monitor github prs"。 这表明Anthropic正在从"会话式工具"向"常驻式服务"转型。

🎯 已知能力
  • 推送通知：主动而非被动响应，具备真正的 proactive 能力
  • GitHub PR监控：代码审查自动化，CI/CD集成
  • Always-On常驻：不依赖用户主动发起会话
  • 云端托管：与Claude Code本地运行不同，KAIROS是云服务
🎁 泄露的其他未发布功能

KAIROS只是冰山一角，泄露代码还显示Anthropic正在开发：

📋 ULTRAPLAN
在远程服务器上生成30分钟Opus会话，规划整个任务
👥 Coordinator Mode
多Agent群体，包含worker和scratchpads
⏰ Agent Triggers
基于cron的定时任务，本质上是CI/CD Agent
🦆 /buddy系统
终端里的电子宠物系统（18个物种，稀有度系统）
🏢 Anthropic的战略意图

KAIROS代表了Anthropic对AI Agent未来的判断：从"工具"向"员工"转型。 与Claude Code需要用户主动打开终端不同，KAIROS是常驻云端、主动服务的Agent。 这与OpenClaw和Hermes的"自托管"路线形成鲜明对比——Anthropic选择了SaaS化的商业模式， 将Agent作为云服务提供，而非软件分发。

🧠 核心洞察与趋势判断
🔍 洞察一：三种架构哲学的分化

三大平台代表了AI Agent的三种不同架构哲学：

学习优先
Hermes押注Agent的自我进化能力。它的假设是：一个能学习的Agent，长期来看会超越一个功能丰富但不会学习的Agent。 这是一种"内功"路线——初期可能不如OpenClaw功能多，但越用越强。
生态优先
OpenClaw押注网络效应。它的假设是：一个拥有13,000个技能的生态系统，比单个Agent再聪明也有价值。 这是"外延"路线——用社区力量弥补个体能力的不足。
托管优先
KAIROS押注云端化和易用性。它的假设是：大多数用户不想自托管，愿意付费换取便利。 这是"SaaS"路线——将Agent作为服务而非软件。
🔮 洞察二：融合而非替代

值得注意的是，这三者不是零和竞争关系。Hermes已经支持从ClawHub安装社区技能， 并提供官方迁移工具；agentskills.io标准旨在让技能跨平台可移植。生产级系统可能会组合三者的优势： OpenClaw的生态广度 + Hermes的学习深度 + KAIROS的云端便利。

⚠️ 洞察三：安全成为分水岭

OpenClaw的ClawHavoc事件和341个恶意技能暴露了生态扩张与安全之间的张力。 Hermes从架构上更保守（容器隔离、预执行扫描），OpenClaw从运营上补救（VirusTotal合作）。 随着Agent获得更高权限（如KAIROS监控PR、发送通知），安全将成为用户选择的首要考量。

🎯 洞察四：从"工具"到"员工"的范式转移

这三个平台的共同点是：它们都在推动AI Agent从"会话式工具"向"常驻式员工"转型。

2024-2025
会话式AI（ChatGPT/Claude）——"我问，你答"
2025-2026
持久化会话（Claude Code/Cursor）——"记住我们聊过什么"
2026+
常驻Agent（Hermes/OpenClaw/KAIROS）——"我在，随时待命"

这个转移类似于Kubernetes生态中"无状态容器→有状态服务"的演进。 当Agent成为长期运行的进程，它们积累的知识（关于你的代码库、工作流程、决策模式） 将成为新的竞争壁垒。

🔮 趋势预测

基于当前格局，我们预测以下趋势将在未来12-18个月内显现：

  • 技能标准化：agentskills.io或类似标准将成为跨平台技能交换的基础协议
  • 混合部署：企业用户将采用"云端+本地"混合模式，敏感任务本地处理，协作任务云端执行
  • Agent市场整合：类似OpenClaw基金会化，更多项目将走向中立治理
  • 安全评级体系：第三方安全审计和技能评分将成为标配
  • 企业级KAIROS：Anthropic可能推出面向企业的KAIROS Enterprise，与自托管方案正面竞争
📝 总结

Hermes Agent、OpenClaw和Anthropic KAIROS代表了AI Agent的三种演进路径： 学习优先、生态优先、托管优先。它们共同指向一个未来——AI Agent将从"你打开的工具"变成"一直在场的协作者"。

选择哪个平台取决于你的核心需求：想要一个会自我进化的Agent？选Hermes。 想要最丰富的生态和最快的上手速度？选OpenClaw。想要零运维的企业级服务？等待KAIROS。 但最可能的结果是：未来的生产系统将是三者的某种组合——生态的广度 + 学习的深度 + 云端的便利。
