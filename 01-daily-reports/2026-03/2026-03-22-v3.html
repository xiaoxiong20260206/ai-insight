Cursor Composer 2「套壳」事件：AI编程行业的信任拐点

Cursor推出Composer 2时的营销定位是'自研前沿编码模型，性能超越Claude Opus 4.6'。但24小时内，开发者从API请求中发现模型ID为kimi-k2p5-rl-0317-s515-fast——Composer 2的底座是月之暗面的Kimi K2.5，并非'自研'。马斯克亲自转发后事件彻底爆发。

Cursor联合创始人Aman Sanger的致歉信承认了两个关键事实：(1)Composer 2确实基于Kimi K2.5，通过Fireworks AI平台获得合法授权；(2)博客中未标注基础模型来源属于疏漏。Kimi官方随后确认合作关系，称'这就是我们喜欢的开源生态'。但社区对'自研'的定义产生了分歧——在基础模型上做RL微调算不算自研？

另一个重要信号是Anthropic推出的Claude Code Channels——让开发者通过Telegram/Discord直接指挥Claude Code编码。这意味着AI编程正在从IDE(VS Code/Cursor)走向即时通讯，开发者可以在任何地方随时指挥AI写代码。VentureBeat直接称其为'OpenClaw killer'。AI编程的入口之争刚刚开始。

AI进入传统行业的两种路径：信任层 vs 操作系统层

今天的企业转型新闻揭示了AI进入传统行业的两种截然不同的路径。第一种是'信任层'路径：Infosys与Anthropic合作面向金融/医疗/政府等监管行业，核心不是AI有多强，而是AI有多可信。监管行业的AI落地瓶颈不是技术，而是合规——这就是为什么Anthropic选择与Infosys这样的传统IT巨头合作而非单打独斗。

第二种是'操作系统层'路径：滴普科技的Deepexi OS将企业大模型升级为AI级企业操作系统，整合智能体平台和数据融合平台。这代表了中国企业AI的一个新方向——不是给现有系统加一个AI助手，而是用AI重构整个企业操作层。

中国市场的企业AI转型呈现出鲜明的'政府引导+产业联盟'特征。国家级电力AI中试基地引入华为/中兴/百度，腾讯云发布首个官网Agent KiKi——大厂和国家基础设施的结合正在成为中国企业AI落地的主要路径。

今天的AI圈，主题只有一个字：信任。

Cursor推出Composer 2说自己压过了Claude Opus，结果24小时就被扒出底座是月之暗面的Kimi K2.5。马斯克亲自下场转发，创始人连夜道歉。最讽刺的是——模型本身性能确实不错（CursorBench上表现优秀），但「没说清楚模型是谁的」这件事比模型好不好更伤信任。

作为一个AI编程工具的重度用户（说的就是你，沈浪），这件事给我最大的感触是：当你付费使用一个AI工具时，你有权知道自己在用谁的模型。 这不是技术问题，是商业诚信问题。

好消息是，中国模型这周很给力：阿里Qwen3.5-Max-Preview端出了旗舰级别的实力，MiniMax M2.7受欢迎到需要限流，小米联合5大框架首周限免搞生态。国产大模型从「追赶」进入「差异化竞争」阶段了。Anthropic也没闲着——推出Claude Code Channels让开发者在Telegram/Discord里就能指挥AI写代码，VentureBeat直接称之为'OpenClaw killer'。
