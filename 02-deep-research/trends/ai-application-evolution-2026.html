📖 研究背景 方法论

AI从"工具"走向"助手"，再走向"Agent"，再到"Agentic"——这条路线人人都能感受到，但很少有人追问：为什么会这样演进？下一阶段又会到哪里？

通常的分析方法是"描述现象+列举产品"，但这样做出来的分析往往是分散的、碎片化的。本次调研换了一种思路：先提炼规律假设，再拿业界一手证据验证规律，最后才基于规律推演未来。

📌 研究流程：假设三条演进规律 → Anthropic/OpenAI/Google官方文档 + 实践者第一手经验交叉验证 → 发现一个反直觉结论 → 基于规律推演下一阶段形态
⚖️ 三大演进规律 已验证

以下三条规律均通过多个独立一手来源交叉验证，置信度：高

R1
认知外包阶梯：结构化认知先于情境化认知被自动化
越容易量化、越有明确成功标准的认知任务，越先被AI接管
人类的认知任务可以按"可量化程度"分层：感知（图像/语音识别）→ 记忆（信息检索/RAG）→ 推理（分析/判断）→ 执行（规划/行动）→ 情境化判断（价值观/隐性约束）。越结构化的层次，越早有明确的自动化指标，也就越先被攻克。
✅ Anthropic 2024：5大Workflow模式中，"提示链"（最结构化）→"评估者-优化者"（需要迭代判断）的顺序，完美映射认知外包阶梯。"Agent"只在"无法预测步骤数"时才用——即任务结构化程度决定使用哪层工具。
✅ OpenAI GPT-5.2 2025：代码任务（SWE-bench Verified 80%）远高于知识工作（GDPval 70.9%）。代码有可运行的客观成功标准，知识工作含有更多情境化判断——规律精确成立。
✅ 类比验证：工业革命中重复性体力劳动（纺织/铸造）先于情境性脑力劳动（设计/管理）被自动化，AI时代以快数倍的速度重演了同一规律。
R2
协调成本决定阶段边界：新协议/标准诞生触发下一阶段跃迁
每次阶段跃迁的前提不是模型更聪明，而是协调机制更便宜
AI能力进化不是线性增强，而是阶跃式跳跃。每次跳跃都伴随着新的协调基础设施出现：工具标准 → 单Agent接口 → 多Agent协议 → 跨组织互联。协调成本降低的那一刻，边际效益突然跃升，一个新阶段就此开始。
✅ Anthropic MCP 2024：工具-AI连接标准出现 → 降低了"给Agent提供工具"的协调成本 → 单Agent阶段加速普及
✅ Google A2A 2025年4月：50+合作伙伴，基于HTTP/SSE/JSON-RPC（复用既有IT协议，降低接入成本）。四大机制：Agent Card（能力发现）+ Task管理 + 实时反馈 + 内容协商——Agent Card就是"降低能力发现成本"的协调协议。
✅ 时序验证：先有MCP（工具层，2024）→ 再有A2A（Agent层，2025）→ 再有AP2支付协议（Agent商业交换层，2025年9月）。协调层级精确跟随演进阶段。
✅ 历史类比：TCP/IP → HTTP → REST API每次标准化都触发下一轮互联网应用爆发，逻辑完全一致
R3
人类审查带宽决定多Agent并行上限
并行规模受限于人类的"审查吞吐量"，而非AI的执行能力
多Agent系统的瓶颈不在AI算力，而在人类的审查带宽：人类单位时间能高质量Review多少AI产出？这个带宽决定了多Agent并行的实际上限，也决定了"可信任的自主度"边界在哪里。
✅ Simon Willison 2025/10："AI-generated code needs to be reviewed, which means the natural bottleneck on all of this is how fast I can review the results."（直接表述）他的结论是：只有"低审查成本"的任务（维护性小任务/研究类任务）才适合真正并行。
✅ Anthropic 2024："Agents are ideal for scaling tasks in trusted environments"——"信任"是"降低审查带宽消耗"的代理变量。越可信的环境，每次审查消耗的带宽越少，多Agent并行规模才能越大。
✅ 系统含义：跨越这一限制的路径有两条——①提高AI产出的可验证性（让审查更快）；②建立AI信誉体系（让信任可累积，减少每次审查的必要性）。两条路业界都在走。

🔬 业界证据链 一手来源

🌿 Anthropic：Workflow vs Agent二分法

来源：📄 Building Effective Agents (2024/12)

Anthropic明确定义两类系统：Workflow（预定义代码路径）和Agent（LLM动态指向自身流程）。这个二分法背后正是规律R1的精确体现——任务是否结构化，决定用Workflow还是Agent。

五大模式（提示链/路由/并行化/编排者-工作者/评估者-优化者）构成了从"低协调成本"到"高协调成本"的完整谱系，验证规律R2。

最重要的洞察："Success isn't about building the most sophisticated system—it's about building the right system." 这是对滥用Agent的直接告诫。

🤖 OpenAI：从多Agent到Mega-Agent的反转

来源：📄 GPT-5.2 (2025/12)   📄 Codex DevDay 2025

GPT-5.2关键数据：SWE-bench Verified 80%（代码，高结构化）；GDPval 70.9%（知识工作，情境化）。两个数字的差距，精确反映规律R1中"结构化 vs 情境化"的进展差异。

Codex的架构选择："We collapsed a fragile, multi-agent system into a single mega-agent with 20+ tools"——这个反转见下方"反直觉发现"专题。

🔗 Google A2A：协调基础设施的诞生

来源：📄 A2A协议 (2025/04)   📄 AP2 (2025/09)

A2A的四个核心机制完美映射规律R2的"协调基础设施"：Agent Card（能力发现）+ Task管理 + 实时状态同步 + 内容格式协商。50+合作伙伴，选择HTTP/SSE/JSON-RPC而非全新协议——正是"降低接入协调成本"的战略选择。

时序证据链：MCP（2024 工具层）→ A2A（2025/04 Agent层）→ AP2（2025/09 商业交换层）——三层协议按照规律R2预测的顺序依次出现。

🛠️ 实践者视角：审查带宽的真实体验

来源：📄 Simon Willison · Parallel Coding Agents (2025/10)

这是规律R3（人类审查带宽）最直接的第一手证据。Simon Willison在使用多个并行编码Agent后的实战结论：

✅ 适合并行的任务：研究探索（无需合并代码）、小维护任务（低审查成本）、精确规格驱动的实现（Spec已确定，审查更快）
❌ 不适合并行的情况：大型新功能（高审查成本）、跨模块重构（协调复杂度高）

他的核心洞察与R3完全一致：并行Agent的效益不来自Agent数量，而来自"任务的审查成本是否足够低"。
🧠 本质洞察 规律总结

🔭 三条规律的统一解释

表面现象
AI从工具 → 助手 → 单Agent → 多Agent → 生态系统，能力在"越来越自主"地演进
本质规律
AI能力的演进本质上是"认知分工"的重新配置：哪些认知任务可以外包、哪些协调成本可以降低、哪些审查带宽可以节省——这三个变量共同决定了每一阶段的边界。
类比生产组织演化：农业社会（人做所有事）→ 工业革命（机器接管可量化劳动）→ 流水线（协调机制降低分工成本）→ 全球供应链（跨组织协议互联）→ AI时代在以超高速度重演同一规律，但速度快100倍。
核心推论
当一个认知任务满足以下三个条件，它就处于"即将被自动化"的临界点：
① 有量化的成功标准（R1：结构化认知）
② 有标准化的执行协议（R2：协调基础设施）
③ 人类审查成本可接受（R3：审查带宽可控）

这三个条件任何一个不满足，该任务就会停留在"人类监督"阶段，而非"自动化"阶段。
对未来的预测
下一个被攻克的大领域是：跨系统的、有明确KPI的企业业务流程——因为这三个条件都在快速成熟（A2A提供协议基础，AI信誉体系降低审查成本，企业数字化提供量化指标）。
⚡ 反直觉发现 重要
⚠️ 多Agent ≠ 一定优于单Agent：OpenAI Codex的反转
OpenAI Codex团队的实际选择："We collapsed a fragile, multi-agent system into a single mega-agent with 20+ tools"。

为什么反转？ 多Agent系统的理论优势（并行+专业化）在实践中被两个隐患抵消：①协调成本（Agent间通信开销）；②错误累积（多Agent链路中错误会级联放大）。

什么时候多Agent才真正有优势？ 并行沙箱执行（OpenAI Codex Cloud：在独立沙箱中并行跑多个任务实例，而非共享状态的多Agent协作）——这是"在物理层面并行"而非"在逻辑层面多角色协作"。

实践结论： 多Agent的真正价值是并行吞吐，而非角色分工。当模型能力足够强时，单Mega-Agent + 多工具比多个弱Agent协作更稳定。模型能力是关键变量，协调架构是次要变量。
维度 结构化多Agent（Team） Mega-Agent + 多工具 分布式Swarm
适用场景 专业化分工，任务边界清晰 模型能力强，任务可在单上下文完成 去中心化探索（实验阶段）
协调成本 中（Orchestrator负责） 低（单上下文内） 高（无中心化协调）
错误传播 中（链路可控） 低（单点，易debug） 高（涌现行为难预测）
业界现状 主流（LangGraph/CrewAI） 当前最优（GPT-5.2 Codex） 研究阶段，生产慎用

基于三大规律外推，而非主观想象

2025–2026
结构化多Agent爆发
以A2A为基础设施，企业内部的Orchestrator-Workers模式快速普及。Gartner预测企业应用中Agent渗透率从<5%升至40%（2026）。
📊 信号：SAP Joule、Salesforce Agentforce、ServiceNow已在生产部署
2026–2027
AI信誉体系初步建立
为突破"审查带宽"瓶颈（R3），Agent需要累积信任记录。类似Stripe的支付信誉或OAuth的授权机制——Agent信誉成为新的基础设施层。
📊 信号：AP2协议已引入Agent身份认证，A2A安全模型对标OpenAPI
2027–2028
情境化认知开始突破
随着模型在情境化任务（GDPval等）接近专家水平，"隐性约束"和"组织语境"理解能力提升，更多之前需要人类判断的软性任务开始可自动化。
📊 信号：GPT-5.2 GDPval 70.9%，还差约25%到全面专家水平
2028+
跨组织Agent协作成常态
不同公司的Agent通过A2A/AP2互相签约、交付服务、结算费用。第一批"AI原生供应链"在数字化程度最高的行业（金融/SaaS）率先出现。
📊 信号：Supertab已演示Agent自主支付，LangChain A2A合作

基于规律给出的实践建议，而非"所有AI都应该用Agent"的空话

你的任务有明确的成功标准（可量化/可自动测试）吗？
是 → 首先考虑用AI自动化；否 → 先设计评估体系，再考虑AI
↓
任务是否需要多步骤协调（≥3步且步骤间有依赖）？
是 → 考虑Workflow/Agent；否 → 单次调用+优化Prompt即可
↓
你能在10分钟内审查完单次AI产出吗？
是 → 可以试用多Agent并行；否 → 先降低单次任务规模，或先建立可验证性机制
↓
是否需要跨系统/跨组织集成？
是 → 关注A2A协议，提前设计Agent Card；否 → MCP工具层已够用
