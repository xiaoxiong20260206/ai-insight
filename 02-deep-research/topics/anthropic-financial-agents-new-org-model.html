02 · Skills+Connectors+Subagents

不是AI工具，是一套组织方式

章节概览
Layer 01
Skills — 岗位手册
领域知识+指令：金融建模规范、合规流程、风控政策可注入。简单的用Markdown写，高级的可执行脚本。金融公司可以把自己的规矩塞进去——Agent变成"按你的规矩干活的新员工"。
模块化能力包
Layer 02
Connectors — 系统权限
受控数据接入：FactSet、S&P Capital IQ、MSCI、PitchBook、Morningstar等16+金融数据源。新增8个（D&B、Fiscal AI等）。Moody's MCP app提供6亿+企业信用评级直接嵌入Claude。
数据接入层
Layer 03
Subagents — 请教专家
主Agent调子模型：可比公司筛选、方法论校验等。不同Agent配置不同Skills——金融分析Agent用风险评估skills，客户支持Agent用CRM skills。层级委托，避免单一Agent认知瓶颈。
层级委托
和小无相功的对照
Skills=SKILL.md，Connectors=工具链/数据接入层，Subagents=人格Agent层级委托——同一思想的不同实现。Anthropic做成商业产品，我们做自进化底层架构。架构思路验证了，但执行速度要快。
Domain-agnostic架构
Reddit/ClaudeCode社区指出：skill files定义触发条件和工作流步骤，connectors接入数据，subagents处理专门子任务——三因子模型在任何行业都能复用。金融只是第一个，法律、医疗、保险会接踵而至。

两种部署模式
Cowork/Code插件模式：Agent与分析师并肩工作，使用桌面已有软件。给Pitch Agent一个目标名单，就能得到Excel可比模型、PowerPoint pitchbook草稿、Outlook覆盖信。
Managed Agent模式：同一个模板在Claude Platform上自主运行，适合处理整本书的交易或夜间排程。Cookbook提供长会话（数小时）、per-tool权限、managed credential vaults、全量audit log。
核心原则：Users stay in the loop
Anthropic反复强调：用户审核、迭代、批准后再提交——Agent不做最终决定。这不是技术限制，是产品设计哲学。金融行业的信任需要逐步建立。
06.5 · Microsoft 365 深度整合

四个场景，一个上下文

章节概览
📊 Excel — 金融建模引擎
从申报资料和数据源自动建立财务模型，审计跨linked workbook的公式，运行敏感性分析。分析师在Excel里直接与Claude交互——不切换窗口。

典型用例：分析师输入目标公司名称，Claude自动从FactSet/S&P拉数据填入模型模板，标记关键假设点，运行3种场景敏感性测试。
生产级
📈 PowerPoint — 自动Pitchbook
起草deck，当底层数据变化时自动更新图表。Excel里建的模型，到PPT自动转成可视化——不需要手动复制粘贴。

典型用例：Pitch builder Agent产出comps表格（Excel）+公司概况页（PowerPoint）+覆盖信草稿（Outlook），一次任务三份产出。
生产级
📝 Word — 信用备忘合规编辑
按公司自有模板编辑信用备忘。Claude理解公司格式要求和审批流程，自动填入数据、检查合规性。

典型用例：Valuation reviewer Agent产出估值报告（Word），按公司内部review standard格式排版，标注需要人工审核的关键假设。
生产级
📬 Outlook — 首席参谋（Coming Soon）
分拣邮件、安排会议、用你的语气草拟回复。会议准备Agent把客户简报+对手方分析推到你的inbox。

典型用例：Meeting preparer Agent把客户背景+最新财报+行业动态打包成会议简报，Outlook提醒你明天10点有call。
Beta
上下文跨应用延续
核心创新不是"Claude能在Excel/PPT/Word里干活"，是上下文自动延续。分析师在Excel里建的模型，不需要到PPT里重新解释——Claude自动理解上下文。这是从"AI辅助每个应用"到"AI串联整个工作流"的质变。
Dispatch — 离开桌面也能指挥
Claude Cowork新增Dispatch功能：通过文字或语音从任何地方分配任务给Claude。分析师离开办公桌后Claude继续处理本地文件，回来时已完成待审核。这是"AI员工"概念的落地——不是需要你盯着才干活。
06.8 · 数据生态：16+ Connector + Moody's MCP

Agent的数据基础设施

章节概览
Connector vs MCP App 的区别
Connector：给Claude governed、实时数据访问——类似员工获得系统权限，只能读数据。
MCP App：更进一步，把数据提供商自己的工具直接嵌入Claude——类似员工不仅有权限，还能用对方的完整工作台。Moody's MCP app不仅提供评级数据，还提供交互式UI，让分析师在Claude里直接操作Moody's的分析界面。

新增8个Connector详解

Dun & Bradstreet — 企业身份验证
全球商业身份验证标准，D-U-N-S® Number体系。Agent在KYC和合规场景下需要"确认我正在和谁打交道"——D&B的Commercial Graph确保Agent基于验证数据运作，产出可审计的确定性结果。
场景：KYC screener Agent用D&B验证实体身份，合规场景要求100%确定性。
合规刚需

场景：Model builder Agent从Fiscal AI获取目标公司最新基本面，自动更新模型假设。

07 · 本质穿透（沈浪视角）

四层递进

章节概览
L2 · 规律层
Agent商业化 = 行业模板 × 数据生态 × 治理保障
不是"更强的chatbot"，是把行业的知识（Skills）× 数据（Connectors）× 流程（Subagents）打包成可部署的单元。三因子模型才是真正的产品形态。
L3 · 映射层
不是卖发动机，是卖整车
通用模型是发动机（需自己组装），Copilot是改装件（增强工具），Anthropic金融Agent是整车——发动机+底盘+内饰+驾照一起交付，客户只需定制化。
L4 · 演变层
短期（3-6月）：法律、医疗、保险垂直模板出现。模板化是规模化路径。
中期（6-18月）：数据供应商角色重新定义——从"数据源"变成"Agent生态一部分"。Moody's MCP app是信号。
长期（18-36月）：合资模式扩散。AI公司→行业数字化转型合伙人。
L5 · 结构性判断：不可逆，不是周期性
AI Agent落地需要行业知识+数据接入+合规治理——这三样只有行业内的公司才有。合资/生态绑定是唯一路径。Anthropic先做了，其他人会跟进。
