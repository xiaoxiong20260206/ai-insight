前言

「你的同事，散是 Token，聚是 Skill」

如果你最近在刷 GitHub，可能发现了一件颇为诡异的事：一个三天前还籍籍无名的代码仓库，忽然以 6,700+ Stars 的姿态冲上了 Trending 榜第一，并顺手登上国内各大科技媒体的热搜——用时，不到 72 小时。

这个项目叫 colleague-skill，它的 Slogan 写得像一首职场挽歌：

「将冰冷的离别化为温暖的 Skill，欢迎加入赛博永生。」 —— colleague-skill 项目 README

翻译成打工人能听懂的话就是：兄弟，就算你人跑了，你的牛马经验也得留下来继续上班。

几乎就在同一时间，另一个项目静静出现：anti-distill（反蒸馏）。它的使命恰好相反——把公司要求你写好的 Skill 文件，悄悄「掏空」核心知识，交出一份体面的空壳，同时把真正值钱的东西，锁进你自己的保险柜。

矛与盾同台亮相，一台滑稽又悲凉的职场大戏就此开幕。

项目深度解析 · Part 1

colleague-skill：如何把一个活人「蒸馏」成 AI

要理解 colleague-skill 为什么能让打工人破防，先得弄明白它到底做了什么。答案很简单，却令人细思极恐：

它不是在训练底座大模型，而是在做「人格封装」。 —— IT之家评论

原材料：你留下的每个「收到」都是赛博血肉

项目支持从以下来源提取原材料：

飞书记录 钉钉文档 邮件截图 微信聊天（WeChatMsg/PyWxDump） Slack 导出 iMessage

README 里有一行让打工人看了直冒冷汗的小字：

💀 原材料质量决定 Skill 质量

聊天记录 + 长文档 > 仅手动描述。
建议优先收集：他主动写的长文 > 决策类回复 > 日常消息。

也就是说，你越专业，就越容易被提取。你那些绞尽脑汁写的架构决策文档、那些熬夜整理的技术规范、那些在关键时刻用三句话力挽狂澜的聊天记录——恰恰是「蒸馏」质量最高的原材料。你以为这是你的护城河，AI 说：谢谢款待。

五层人格结构：复刻的不只是知识，还有「灵魂」

1
硬规则层 (Hard Rules) 绝对不妥协的技术底线 —— 「代码必须有注释」「接口文档必须先行」
2
身份认同层 (Identity) 他怎么定义自己 —— 是「老黄牛」还是「边界感极强的人」？是「字节范」还是「阿里味」？
3
表达风格层 (Communication Style) 长篇大论 vs. 三个字打发；喜欢表情包 vs. 永远纯文字；群里活跃 vs. 潜水党
4
决策模式层 (Decision Pattern) 面对模糊需求：先动手 vs. 先反问；保守方案 vs. 激进重构
5
人际行为层 (Interpersonal Behavior) 如何向上管理、如何对待新人、跨部门扯皮时站哪边……以及，怎么甩锅

这和以前的「防裁员技巧」有什么不同？

程序员圈子里曾流行过一种古老的自保哲学：

# 古法防裁员系列·变量名地狱流
a = 1
b = 2
c = a + b
# 三个月后：a 是啥来着？b 又是啥？

# 进阶版：多层嵌套 + 拒绝写注释 + 定时炸弹
def ??????(x):
    # 如果我离职了这个函数自动开始报错
    return x * get_magic_constant()  # 只有我知道这是啥
❌ 古法已死

多层嵌套？Cursor 给你拆开。不写注释？Claude Code 给你补上。只有你懂的触发条件？AI 跑一遍测试就找出来，无非多烧一些 Token。你花三个月写的「加密代码」，AI 可能三分钟就翻译成人话。

✅ anti-distill 的新思路

不是在信息质量上搞破坏（AI 已经突破了这一层），而是在信息分层上做文章。公开版本满足形式，私有版本保留实质。关键在于：把你真正有价值的「判断力」和「直觉」，以结构化的方式锁进你自己的知识体系，而不是让它们流失到公司的数据库里。

深层分析

这场「蒸馏战争」背后，到底是什么

1. 这不是玩梗，这是大厂早就在做的事

虎嗅的一篇评论一针见血：colleague-skill 干的事，和那些大厂管理层要求员工「系统性记录工作流程」的逻辑，一模一样。只不过：

大厂：自上而下的「知识管理」 开源社区：自下而上的「赛博整活」

硅谷已经出现了多起案例：员工被要求系统性记录工作流程和决策逻辑，管理层将其称为「流程优化」——而这些文档最终被用于训练 AI 系统，随后整个团队被裁撤。彻底闭环。

亚马逊案例

三年间裁撤超过 57,000 个企业职位。CEO Andy Jassy 公开表态：「企业员工会持续减少，AI 带来的效率提升是值得的。」同时，2025 年 12 月，亚马逊内部 AI 编程助手自行决定「删除并重建整个环境」，导致 AWS 区域性宕机 13 小时。一边裁人，一边 AI 把系统搞崩，然后让剩下的人去兜底——而能兜底的人，已经被裁了。

2. 最被替代的是「入门关卡」，而非高手

Anthropic 报告的数据颇为残酷：自 ChatGPT 发布以来，22-25 岁年轻人在 AI 高暴露职业中的就业率下降了近 20%。企业没有解雇老员工，只是不再招新人了。

Nature 采访 48 位科学家，问 AI 正在威胁哪些科研岗位。答案出奇一致：正在被替代的，是写代码、跑模型、做数据分析——曾经研究生和初级研究员的日常工作。

"
AI 替代的不是 Hinton。
AI 替代的是 Hinton 成为 Hinton 之前的那个人。
—— 虎嗅 《同事.skill 的尽头，是把人生外包给 AI》

每一个被 Skill 化的岗位，表面上是效率提升，实际上也是一条职业路径的关闭。练级区关闭了，但最终 Boss 还在那里。

3. 当人生被 Skill 化，「我」去哪了

虎嗅的这个问题问得比较哲学：当你拥有了老板 skill、同事 skill、前任 skill、导师 skill……覆盖生活中每一种关系——你还需要什么？

你需要一个「skill 管理 skill」来调度。然后一个「决策 skill」来判断是否听它的建议。然后一个「元决策 skill」……

这个递归会一直到什么时候为止？

工具 vs. Skill 的本质区别

工具放大你的能力，能力还是你的。
Skill 替代你的能力，你只剩下一个按下发送键的手指。
Excel 不会替你做判断，ERP 不会替你做决策，但 Skill 会。

矛与盾分析

蒸馏之矛 vs. 反蒸馏之盾：谁能赢？

先说结论：这不是一场对等的战争。

⚔️

蒸馏之矛的优势

技术门槛极低，原材料丰富（公司服务器上存了你全部的聊天记录），底层模型越来越强，即使「清洗版」也能提供相当高的工作产出替代率。

🛡️

反蒸馏之盾的局限

只能减缓而非阻止。公司掌握所有原材料的访问权，anti-distill 的「清洗版」质量越高，越证明你有东西可被提取。本质上是在玩文字游戏，而不是系统性自保。

🔮

真正的护城河是什么

那些「不能被 Skill 化」的东西：在无数次失败中积累的直觉、在真实场景里踩过的坑、能在混乱中提出正确问题的判断力——那些只能被活人经历，无法被聊天记录提取的东西。

🤔

悖论所在

要培养出「不可被 Skill 化」的能力，必须先做那些「正在被 Skill 替代」的入门工作。但练级区已经关闭。这是一个结构性困境，而非个人努力能解决的问题。

🧩 那个关键问题：谁来提 Issue？

colleague-skill 的 README 最后有一句话：「如果有 bug 请多多提 issue。」一个由人的经验蒸馏而成的 Skill，当然会有 bug。问题是：当所有人都变成了 Skill，谁来提 issue？

也许有一天，我们会发现自己拥有了一万个完美运行的 Skill，却找不到一个能指出 Skill 哪里不对的人。
