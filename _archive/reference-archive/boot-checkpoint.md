# 启动检查点模板

> **本文件定义技能加载后必须输出的启动检查点。**
> 
> 目的：防止"加载但不执行启动协议"的问题。

---

## 强制输出格式

当检测到以下任一条件时，必须立即输出启动检查点：

```
条件1: 本技能 SKILL.md 中有 requires: [meta-execution]
条件2: 本技能匹配 binding-list.md 中任意一项
条件3: 用户说"跑一下AI日报"、"跑一下AI周报"等触发词
```

---

## 启动检查点输出模板

```markdown
🚀 [Meta-Execution] 启动检查 | 技能: sl-ai-insight

📋 踩坑经验已召回:
- #74: KIM API返回错误≠消息未投递
- #75: 日期范围算错
- #76: 重复preview
- #77: 未确认就发群
- #78: Git未推送
- #79: 日报索引缺失

⚠️ 关键门控:
- preview仅执行一次
- 必须等待用户确认「发群」
- API错误后不立即重试
- 四位置验证完成后才能交付
```

---

## 禁止行为

| 错误行为 | 正确行为 |
|----------|----------|
| 加载技能后直接进入任务 | 先输出启动检查点，再进入任务 |
| 知道有启动协议但跳过 | 必须执行，不可跳过 |
| 输出激活标记但没检索踩坑 | 必须检索并列出相关编号 |

---

## 检查点通过条件

```
✅ 启动检查点通过 = 
    已输出启动标记 
    AND 已检索踩坑记忆 
    AND 已列出相关编号
    AND 已提示关键门控
```

**只有全部满足，才能进入任务执行。**
