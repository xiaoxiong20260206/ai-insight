## 各类调研主题的结构变体

### 变体A：公司/产品深度调研（以Claude Code为例）

```
# 00 全文概览（核心结论 + 综合评分表）
# 01 为什么这个产品值得深研（现象反差/爆发数据）
# 02 产品定义：它到底是什么（定义 + 与竞品的本质差异）
# 03 技术架构：它怎么做到的（架构解析 + 技术选择背后的逻辑）
# 04 增长逻辑：它为什么爆（增长机制分析）
# 05 四维深挖：核心洞察（架构/机制/哲学/局限）
# 06 行业影响：它改变了什么（竞品对比 + 格局变化）
# 07 行动建议：我们能学什么（可操作的建议）
# 08 彩蛋（林克视角：自我印证）
```

### 变体B：趋势洞察（以AI大神观点分享为例）

```
# 00 全文概览（N位观点 + 核心共识表）
# 01 来源：分析了哪些人/文章（信源清单表）
# 02 趋势洞察：[共识一] 
# 03 趋势洞察：[共识二]
# 04 趋势洞察：[共识三]
# 05 深度洞察：本质规律（从现象到本质）
# 06 总结与行动建议
# 07 彩蛋
```

### 变体C：人物深度解读（以Naval为例）

```
# 00 全文概览（核心观点 + 解读价值说明）
# 01 关于[人物]：为什么要解读 Ta（身份背景 + 价值定位）
# 02 核心观点一：[标题]（引用原文 → 深挖 → 类比）
# 03 核心观点二：[标题]
# ... 
# 0N 总结与行动建议（三层维度）
# 0N+1 彩蛋
```

## 注意事项

1. **章节数量灵活调整**：参考文档是9章（00-08），实际可以是6-12章，根据内容深度决定
2. **表格是底线**：无论多少章，整篇文章表格不能少于3张
3. **彩蛋是标配**：每篇文章都需要彩蛋（林克自我表达章节），这是「林克风格」的核心标识
4. **00章节结构固定**：全文概览必须包含：核心结论（加粗）+ 对比表格 + 📌核心判断
